# LETUS PMS — 실행·배포 안내서

저(Claude)랑 만든 실제 앱 코드예요. Figma Make 없이, 이 코드가 정훈님 Supabase에 직접 연결돼요.
파일: `letus-pms-app.zip` (압축 풀면 `letus-pms` 폴더)

---

## 한눈에 보는 순서
1. Supabase에 쓰기 권한 SQL 실행 (`letus_rls_writes.sql`)
2. Supabase에서 anon key 복사 → 코드에 붙여넣기
3. GitHub에 코드 올리기
4. Vercel에 연결해서 배포 → 라이브 주소 나옴
5. 가입 → 본인을 관리자로 → 사용 / 폰 홈화면 추가

---

## 1. 쓰기 권한 SQL 실행 (꼭 먼저)
앱이 Supabase에 "저장"하려면 권한이 필요해요(지금은 읽기만 됨).
Supabase 대시보드 → SQL 편집기 → `letus_rls_writes.sql` 내용 붙여넣고 **Run**.
(이미 테이블 만드는 마이그레이션은 했으니, 이건 쓰기 권한만 추가하는 거예요.)

## 2. anon key 넣기
- Supabase 대시보드 → 좌측 **Project Settings(설정)** → **API**
- "Project API keys"의 **anon public** 키를 복사
- 압축 푼 폴더에서 `src/supabase.js` 파일을 열어, 아래 줄의 따옴표 안에 붙여넣기:
  ```js
  const SUPABASE_ANON_KEY = "여기에_anon_public_키_붙여넣기";
  ```
  (URL은 이미 정훈님 프로젝트로 채워져 있어요. anon key는 공개돼도 안전한 키예요.)

## 3. GitHub에 올리기
- github.com 가입(무료) → 우측 상단 **+ → New repository** → 이름 `letus-pms` → Create
- 새 저장소 화면에서 **"uploading an existing file"** 클릭
- 압축 푼 `letus-pms` 폴더 **안의 파일들**(`src`, `public`, `package.json`, `index.html`, `vite.config.js` 등)을 드래그해서 업로드 → Commit
  ※ `node_modules`는 올리지 않아요(어차피 없어요). `package.json`은 꼭 포함.

## 4. Vercel 배포
- vercel.com 가입(무료, **GitHub로 로그인**하면 편함)
- **Add New → Project** → 방금 만든 `letus-pms` 저장소 **Import**
- Framework는 **Vite**로 자동 감지됨 → 그대로 **Deploy**
- 1~2분 뒤 **라이브 주소**(예: `letus-pms.vercel.app`)가 나와요. 그게 정훈님 앱이에요.

## 5. 가입 → 관리자 → 사용
- 라이브 주소 접속 → **신규 가입**(본인 이메일/비밀번호)
- 로그인이 안 되면: Supabase → **Authentication** → 이메일 설정에서 **"Confirm email" 끄기**(테스트용)
- 본인을 관리자로 만들기 — Supabase SQL 편집기에 (본인 이메일로 바꿔서):
  ```sql
  insert into user_roles (user_id, role)
  select id, '관리자' from auth.users where email = '본인이메일@example.com'
  on conflict (user_id) do update set role = '관리자';
  ```
- 다시 로그인 → 관리자로 사용

## 폰에서 앱처럼 쓰기 (PWA)
- 폰 브라우저로 라이브 주소 접속 → 공유/메뉴 → **"홈 화면에 추가"**
- 홈에 LETUS 아이콘 생기고, 누르면 주소창 없이 앱처럼 전체화면으로 떠요.
